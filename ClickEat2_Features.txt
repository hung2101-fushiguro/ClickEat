================================================================================
1. User Authentication & Profile
================================================================================
Cung cấp chức năng Đăng ký, Đăng nhập và quản lý thông tin tài khoản cơ bản của người dùng.

a. Class Diagram
[Web Browser] ---> [LoginServlet / RegisterServlet]
                        |
                        v
                  [UserDAO] <--- [AbstractDAO]
                        |
                        v
                  [Database (Users)]

b. Class Specifications

UserDAO Class
Lớp xử lý các thao tác truy xuất dữ liệu liên quan đến bảng Users.
No  Method          Description
01  checkLogin      Input: phone/email, password. Output: Object User. 
                    Xử lý: Kiểm tra sự tồn tại của user, verify password (hỗ trợ hash và plain text).
02  insert          Input: Object User. Output: int (new generated id).
                    Xử lý: Thêm bản ghi mới vào bảng Users với role CUSTOMER mặc định.
03  checkPhoneExist Input: String phone. Output: boolean.
                    Xử lý: Kiểm tra số điện thoại đã được đăng ký chưa.
04  changePassword  Input: userId, newPasswordHash. Output: boolean.
                    Xử lý: Cập nhật mật khẩu mới cho người dùng.

LoginServlet Class
Lớp điều hướng và xử lý logic đăng nhập.
No  Method          Description
01  doPost          Input: request (username, password, remember). Output: redirect.
                    Xử lý: Gọi UserDAO.checkLogin, khởi tạo Session, xử lý cookie "Remember Me".
02  redirectByRole  Input: user object. Output: redirect path.
                    Xử lý: Điều hướng người dùng về trang Dashboard tương ứng với Role.

Database Queries
- SELECT * FROM Users WHERE (phone = ? OR email = ?)
- INSERT INTO Users (full_name, email, phone, password_hash, role, status, created_at) VALUES (?, ?, ?, ?, ?, ?, GETDATE())
- UPDATE Users SET password_hash = ?, updated_at = GETDATE() WHERE id = ?

2. Merchant Menu Management
================================================================================
Cho phép chủ quán quản lý danh mục và các món ăn trong thực đơn của mình.

a. Class Diagram
[Merchant Web] ---> [MerchantCatalogServlet]
                           |
                 ---------------------
                 |                   |
          [CategoryDAO]        [FoodItemDAO]
                 |                   |
                 v                   v
            [Categories]         [FoodItems]

b. Class Specifications

FoodItemDAO Class
Lớp xử lý dữ liệu cho các món ăn.
No  Method          Description
01  findByMerchant  Input: merchantUserId. Output: List<FoodItem>.
                    Xử lý: Lấy danh sách món ăn kèm theo tên shop và tên danh mục.
02  insert          Input: Object FoodItem. Output: int.
                    Xử lý: Thêm món ăn mới vào hệ thống.
03  toggleStatus    Input: itemId, merchantId, isAvailable. Output: boolean.
                    Xử lý: Bật/Tắt trạng thái kinh doanh của món ăn.
04  update          Input: Object FoodItem. Output: boolean.
                    Xử lý: Cập nhật thông tin chi tiết món ăn (tên, giá, mô tả, ảnh).

MerchantCatalogServlet Class
Lớp Controller quản lý giao diện thực đơn chủ quán.
No  Method          Description
01  doGet           Input: request. Output: forward to catalog.jsp.
                    Xử lý: Lấy danh sách Category và FoodItem của merchant để hiển thị.
02  doPost          Input: action (add, edit, toggle). Output: redirect.
                    Xử lý: Điều phối các yêu cầu thay đổi thực đơn thông qua các DAO.

Database Queries
- SELECT fi.*, mp.shop_name AS merchant_name, c.name AS category_name FROM FoodItems fi INNER JOIN Categories c ON c.id = fi.category_id WHERE fi.merchant_user_id = ?
- UPDATE FoodItems SET is_available = ?, updated_at = SYSUTCDATETIME() WHERE id = ? AND merchant_user_id = ?
- INSERT INTO FoodItems (merchant_user_id, category_id, name, description, price, image_url, is_available, is_fried) VALUES (?, ?, ?, ?, ?, ?, ?, ?)

3. Shopping Cart & Order Placement
================================================================================
Xử lý quy trình chọn món, giỏ hàng và đặt hàng cho khách hàng và khách vãng lai.

a. Class Diagram
[Customer/Guest] ---> [CartServlet] ---> [CheckoutServlet]
                           |                   |
                    [CartDAO/ItemDAO]     [OrderDAO/ItemDAO]

b. Class Specifications

CartServlet Class
Quản lý các thao tác trên giỏ hàng.
No  Method          Description
01  doGet (add)     Input: food id. Output: redirect back.
                    Xử lý: Kiểm tra món ăn, lấy/tạo Cart (Session cho Guest), thêm vào CartItems.
02  doGet (view)    Input: none. Output: forward to cart.jsp.
                    Xử lý: Hiển thị danh sách món trong giỏ và tính tổng tiền.

CartDAO Class
Quản lý thực thể Cart trong CSDL.
No  Method          Description
01  getActiveCartByCustomerId Input: customerId. Output: Object Cart.
                    Xử lý: Tìm giỏ hàng có status 'ACTIVE' của người dùng.
02  createGuestCart Input: guestId. Output: int.
                    Xử lý: Tạo giỏ hàng mới cho khách vãng lai dựa trên Client UUID.

OrderDAO Class (Placement Part)
Xử lý lưu trữ đơn hàng.
No  Method          Description
01  insertOrderFull Input: Order record, List<OrderItem>. Output: boolean.
                    Xử lý: Dùng Transaction để lưu đồng thời Order và danh sách OrderItems.

Database Queries
- SELECT * FROM Carts WHERE customer_user_id = ? AND status = 'ACTIVE'
- INSERT INTO Carts (guest_id, merchant_user_id, status) VALUES (?, NULL, 'ACTIVE')
- INSERT INTO OrderItems (order_id, food_item_id, item_name_snapshot, unit_price_snapshot, quantity) VALUES (?, ?, ?, ?, ?)

4. Merchant Order Management
================================================================================
Quy trình xử lý đơn hàng của chủ quán (Nhận đơn -> Chuẩn bị -> Sẵn sàng).

a. Class Diagram
[Merchant Dashboard] ---> [MerchantOrderServlet]
                                |
                           [OrderDAO] ---> [OrderStatusHistory]

b. Class Specifications

MerchantOrderServlet Class
Xử lý luồng trạng thái đơn hàng tại quán.
No  Method          Description
01  doGet           Input: tab status. Output: list of orders.
                    Xử lý: Lọc danh sách đơn hàng theo trạng thái (Pending, Preparing, etc.)
02  doPost          Input: action (accept, ready, cancel). Output: redirect.
                    Xử lý: Gọi DAO để chuyển đổi trạng thái đơn hàng.

OrderDAO Class (Management Part)
Cập nhật trạng thái và lịch sử.
No  Method          Description
01  transitionMerchantOrderStatus Input: orderId, merchantId, newStatus, note. Output: boolean.
                    Xử lý: Cập nhật Orders.order_status và thêm bản ghi vào OrderStatusHistory.

Database Queries
- SELECT * FROM Orders WHERE merchant_user_id = ? AND order_status IN (...) ORDER BY created_at DESC
- UPDATE Orders SET order_status = ?, updated_at = GETDATE() WHERE id = ? AND merchant_user_id = ?
- INSERT INTO OrderStatusHistory (order_id, from_status, to_status, updated_by_role, updated_by_user_id, note) VALUES (?, ?, ?, 'MERCHANT', ?, ?)

5. Shipper Delivery Workflow
================================================================================
Quy trình nhận đơn, định vị và xác nhận giao hàng của Shipper.

a. Class Diagram
[Shipper App] ---> [ShipperDashboardServlet] ---> [ShipperProofServlet]
                          |                             |
                      [OrderDAO]                  [ShipperDAO]

b. Class Specifications

ShipperDashboardServlet Class
Trang chủ của shipper để tìm và nhận đơn.
No  Method          Description
01  doGet           Input: none. Output: dashboard.jsp.
                    Xử lý: Lấy danh sách đơn 'READY_FOR_PICKUP' và đơn shipper đang nhận.

OrderDAO Class (Shipper Part)
Xử lý nhận đơn và hoàn thành.
No  Method          Description
01  shipperClaimOrder Input: orderId, shipperId. Output: boolean.
                    Xử lý: Gán shipper_user_id cho đơn và chuyển sang trạng thái DELIVERING.
02  completeDelivery  Input: orderId, proofUrl. Output: boolean.
                    Xử lý: Cập nhật trạng thái DELIVERED và lưu ảnh bằng chứng giao hàng.

Database Queries
- SELECT * FROM Orders WHERE order_status = 'READY_FOR_PICKUP' AND shipper_user_id IS NULL
- UPDATE Orders SET shipper_user_id = ?, order_status = 'DELIVERING', picked_up_at = GETDATE() WHERE id = ?
- UPDATE Orders SET order_status = 'DELIVERED', delivered_at = GETDATE(), proof_image_url = ? WHERE id = ?

6. AI Chat Enhancement
================================================================================
Tư vấn dinh dưỡng và gợi ý món ăn tự động dựa trên lịch sử người dùng qua Gemini AI.

a. Class Diagram
[Customer UI] ---> [AIChatServlet] <---> [Google Gemini API]
                        |
                  [OrderDAO] (Dữ liệu bối cảnh)

b. Class Specifications

AIChatServlet Class
Cầu nối giữa người dùng và trí tuệ nhân tạo.
No  Method          Description
01  doPost          Input: userMessage. Output: AI Reply.
                    Xử lý: Tổng hợp Food History 14 ngày, nhồi System Instruction và gọi callGeminiAPI.
02  callGeminiAPI   Input: prompt. Output: text response.
                    Xử lý: Gửi request JSON tới REST API của Gemini và parse kết quả trả về.

OrderDAO Class (AI Support)
Cung cấp dữ liệu ngữ cảnh cho AI.
No  Method          Description
01  getCustomerFoodHistory Input: customerId, days. Output: String (text list).
                    Xử lý: Truy vấn các món khách đã ăn trong quá khứ để AI phân tích thói quen.
02  getAvailableMenuContext Input: none. Output: String.
                    Xử lý: Lấy danh sách toàn bộ món đang bán để AI gợi ý chính xác.

Database Queries
- SELECT i.item_name_snapshot, i.quantity, o.created_at FROM Orders o JOIN OrderItems i ON o.id = i.order_id WHERE o.customer_user_id = ? AND o.created_at >= ...
- SELECT fi.name, fi.price, mp.shop_name FROM FoodItems fi JOIN MerchantProfiles mp ON fi.merchant_user_id = mp.user_id WHERE fi.is_available = 1
